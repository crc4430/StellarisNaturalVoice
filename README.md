# Stellaris Natural Voice

Give Stellaris' in-game **"read aloud" (speaker / 🔊) button** a **natural,
human-sounding neural voice** instead of the robotic built-in Microsoft David /
Zira / Mark voices.

Stellaris speaks through the default Windows **SAPI5** voice. This project
installs a tiny native SAPI5 voice that streams **Microsoft Azure neural
text-to-speech** — so the narrator sounds like a real person. It also works in
any other classic SAPI app (Balabolka, NVDA, TTS readers, etc.).

- ✅ Natural neural voices (Thomas 🇬🇧, Christopher / Aria / Guy / Jenny 🇺🇸)
- ✅ Free for normal use (Azure **F0** tier = 500,000 characters/month, $0)
- ✅ No AI model on your PC — a ~1.5 MB DLL; audio is generated in the cloud
- ✅ Your Azure key is stored only in your Windows registry, never in a file
- ✅ Adjustable speed, switchable voices, one-command fixes
- ⚠️ Needs an internet connection (each sentence is a quick cloud request)

> **Why not a fully offline voice?** Stellaris only plays Windows SAPI5 voices,
> and the genuinely natural offline engines (Piper, Kokoro, etc.) aren't exposed
> to SAPI5 in a way Stellaris will load. Azure neural over a thin SAPI bridge is
> the most reliable way to get a natural voice into the game today.

## Requirements

- **Windows 10 or 11** (64-bit).
- A free **Microsoft Azure** account (used to generate the speech — see Step 1).
- An internet connection while playing.
- Stellaris (any recent version). No mods required.

---

## Step 1 — Get a free Azure Speech key

The voice audio is generated by Azure's neural TTS. The **Free F0 tier** gives
500,000 characters per month at **no charge** — far more than you'll use reading
event pop-ups. (Creating an Azure account asks for a card for identity
verification; the F0 tier is not billed.)

1. Go to **https://azure.microsoft.com/free** and create/sign in to an account.
2. Open the **Azure Portal** → **Create a resource** → search **"Speech"**
   (by Microsoft, under Azure AI services) → **Create**.
3. Fill in:
   - **Subscription:** your free subscription
   - **Resource group:** create one (any name, e.g. `tts`)
   - **Region:** pick one near you (e.g. `eastus`, `westeurope`) — **remember this**
   - **Name:** any name (e.g. `stellaris-tts`)
   - **Pricing tier:** **Free F0** *(important — this is the $0 tier)*
4. Click **Review + create** → **Create** and wait for deployment.
5. Open the resource → left menu **Keys and Endpoint**.
6. Copy **KEY 1** and the **Location/Region** (e.g. `eastus`). You'll paste these
   during install.

> If "Free F0" is greyed out, you may already have an F0 Speech resource on the
> subscription (only one is allowed). Reuse it, or use a Standard **S0** resource
> (pay-as-you-go, ~$15 per 1,000,000 characters — pennies for this use).

## Step 2 — Install

### Option A — Download the prebuilt release (easiest)

1. Download **`StellarisNaturalVoice-win-x64.zip`** from the
   [**Releases**](https://github.com/crc4430/StellarisNaturalVoice/releases) page.
2. Extract it anywhere (e.g. `Documents\StellarisNaturalVoice`). It contains
   `setup.exe` and `azure_sapi.dll` — keep them together.
3. Open **PowerShell** in that folder and run:

   ```powershell
   # Recommended: pass your key/region via environment variables
   $env:AZURE_SPEECH_REGION = "eastus"          # your region from Step 1
   $env:AZURE_SPEECH_KEY    = "<paste KEY 1>"    # your key from Step 1
   .\setup.exe install
   ```

   Or just double-click `setup.exe` (or run `.\setup.exe install`) and answer the
   two prompts for region and key.

> Windows **SmartScreen** may warn that `setup.exe` is from an unknown publisher
> (it's an unsigned binary). Click **More info → Run anyway**. The binary is built
> from the source in this repo — you can also build it yourself (Option B).

### Option B — Build from source

See [Build from source](#build-from-source). Requires the Rust toolchain; the
result is the same `setup.exe` + `azure_sapi.dll`.

### What install does

- Registers the neural voices for your user account (**no admin needed**).
- Saves your region/key onto the voice's registry entry (not in any file).
- Sets **Thomas** as the default voice and **clears the OneCore override** that
  would otherwise stop Stellaris from using it (see the
  [important note](#-if-the-voice-reverts-to-robotic-important) below).
- Speaks a test sentence so you can confirm it works.

## Step 3 — Use it in Stellaris

1. **Fully close Stellaris** if it's open (it picks the voice at launch).
2. Launch Stellaris.
3. Open any event window / tooltip and click the **speaker (🔊) icon**.

You should hear the natural voice read the text. There's no in-game setting to
change — Stellaris just uses your Windows default voice, which is now this one.

> Make sure **Text-to-Speech is enabled** in Stellaris:
> *Settings → Accessibility* (the general/tooltip TTS toggles).

## Switch voice

Run from the folder containing `setup.exe`:

```powershell
.\setup.exe default-voice AzureThomas        # 🇬🇧 UK male (default)
.\setup.exe default-voice AzureChristopher   # 🇺🇸 US deep male
.\setup.exe default-voice AzureAria          # 🇺🇸 US warm female
.\setup.exe default-voice AzureGuy           # 🇺🇸 US natural male
.\setup.exe default-voice AzureJenny         # 🇺🇸 US friendly female
```

Then **restart Stellaris**. Preview a voice without the game:

```powershell
.\setup.exe test                 # speaks the current default voice
.\setup.exe test AzureAria       # speaks a specific voice
```

Want a different Azure voice (hundreds exist)? Edit the `VOICES` list in
`azure-sapi/src/bin/setup/registry.rs` and rebuild.

## Change speed

```powershell
.\setup.exe rate 1.0    # normal
.\setup.exe rate 1.1    # 10% faster
.\setup.exe rate 1.3    # 30% faster
.\setup.exe rate 0.9    # 10% slower
```

Range is **0.5–2.0**. Applies to all voices. **Restart Stellaris** for it to take
effect.

## ⚠️ If the voice reverts to robotic (important)

Stellaris reads the **modern Windows "default speech voice"** stored at:

```
HKCU\SOFTWARE\Microsoft\Speech_OneCore\Settings\TextToSpeech\Voice
```

**Opening the Speech control panel (`sapi.cpl`) or Windows Settings → Time &
Language → Speech, and selecting a voice, writes that value to a Microsoft voice
— which overrides this tool and makes Stellaris sound robotic again.**

**Fix it in one command:**

```powershell
.\setup.exe default-voice AzureThomas   # or whichever voice you want
```

…then fully restart Stellaris. That command re-clears the override so Stellaris
falls back to this tool's voice.

**To avoid it:** don't pick a voice in those Windows Settings screens while using
this tool. You can verify the override is clear (empty = good):

```powershell
(Get-ItemProperty 'HKCU:\SOFTWARE\Microsoft\Speech_OneCore\Settings\TextToSpeech' -Name Voice -ErrorAction SilentlyContinue).Voice
```

## Update your Azure key

If your key changes (or you switch Azure resources), re-run install with the new
values — it rewrites the saved key/region:

```powershell
$env:AZURE_SPEECH_REGION = "eastus"
$env:AZURE_SPEECH_KEY    = "<new key>"
.\setup.exe install
```

## Uninstall

```powershell
.\setup.exe uninstall          # remove the voices and registry entries
.\setup.exe uninstall --purge  # also delete %LOCALAPPDATA%\AzureSapi (DLL + log)
```

This restores Windows to its normal voices. Restart Stellaris afterward.

## Troubleshooting

**Where to look:** every request and error is logged to
`%LOCALAPPDATA%\AzureSapi\engine.log`.

| Symptom | Cause & fix |
|---|---|
| **Robotic voice in-game** | The OneCore override got set (usually by opening a Speech settings screen). Run `.\setup.exe default-voice <Voice>` and restart Stellaris. See [the note above](#-if-the-voice-reverts-to-robotic-important). |
| **A short low beep instead of speech** | Synthesis failed — check `engine.log`. See the specific errors below. |
| `Azure HTTP 401` in the log | Wrong key, or key/region mismatch. Re-run install with the correct **KEY 1** and **Region** from the Azure portal. |
| `Azure HTTP 429` in the log | You hit the monthly F0 quota (500k chars). Wait for the monthly reset, or move the resource to S0. |
| `WinHttp...` / no response | No internet, or a firewall/proxy is blocking `*.tts.speech.microsoft.com`. |
| **No sound at all in Stellaris** | Confirm TTS is on in *Stellaris → Settings → Accessibility*, and that you fully restarted the game after install. |
| **Nothing happens / wrong voice** | Verify the default: `(New-Object -ComObject SAPI.SpVoice).Voice.GetDescription()` should print one of the `Azure ...` voices. If not, run `.\setup.exe default-voice <Voice>`. |
| **Slight delay before the first words** | Normal — each sentence is a quick cloud round-trip. Audio starts after the first sentence is synthesized. |
| **SmartScreen blocks setup.exe** | "More info → Run anyway" (unsigned binary), or build it yourself. |

Still stuck? Open `engine.log`, reproduce the issue in-game, and check the newest
lines — they name the exact failure.

## Cost / free tier

- **Free F0 tier:** 500,000 characters/month, **$0**, resets monthly. A typical
  Stellaris event pop-up is a few hundred characters, so this covers very heavy
  use. If you exceed it on F0, Azure simply returns an error until reset — it does
  **not** auto-charge you.
- **Standard S0 tier:** pay-as-you-go, ~**$15 per 1,000,000 characters** of neural
  TTS. For click-to-read use that's a few cents. Useful if you can't create an F0
  resource.
- Only text you actually trigger is sent (and counted). Optional: set a small
  **Budget alert** in *Azure Portal → Cost Management* for peace of mind.

## Privacy & security

- Your Azure key is written **only** to your Windows registry
  (`HKCU\...\Speech\Voices\Tokens\Azure*`), never to a file in this project.
- The text you click to read is sent over HTTPS to Azure to synthesize audio
  (that's how cloud TTS works). Nothing else is collected; there is no telemetry.
- The engine makes requests only to `https://<region>.tts.speech.microsoft.com`.
- Network/TLS uses the OS stack (WinHTTP). No bundled certificates or model files.

## How it works

```
Stellaris speaker (🔊) button
  -> Windows SAPI5  (default voice = Azure Thomas/Christopher/...)
  -> azure_sapi.dll  (ISpTTSEngine, loaded in-process by Windows)
  -> HTTPS POST SSML -> https://<region>.tts.speech.microsoft.com   (WinHTTP, OS TLS)
  <- 24 kHz 16-bit mono PCM
  -> played back to Stellaris through SAPI
```

The DLL is a native COM in-process SAPI5 TTS engine. When the host asks it to
speak, it splits the text into sentences, sends each to the Azure REST endpoint
as SSML (with your chosen voice and speed), and streams the returned PCM back to
SAPI — including word-boundary events and pause/stop handling.

Source layout:

```
azure-sapi/
  src/
    lib.rs            COM server (DllGetClassObject, class factory)
    engine.rs         ISpTTSEngine: sentence split, pacing, word events, PCM write
    azure.rs          Azure REST call over WinHTTP, SSML build, PCM decode
    config.rs         log/assets dir resolution
    logger.rs         file logger (engine.log)
    bin/setup/
      main.rs         install / uninstall / default-voice / rate / test CLI
      registry.rs     COM + voice token registration, default voice, speed
      speak.rs        the smoke-test that speaks through SAPI
build.ps1             one-shot build + stage to dist\
```

## Build from source

Building needs the Rust toolchain. On a machine **with Visual Studio Build Tools**
the default MSVC toolchain works. This project was developed on a machine **without
MSVC**, using Rust's **GNU** toolchain plus WinLibs MinGW-w64:

```powershell
# 1. Install Rust (https://rustup.rs). If you use the GNU host, also install MinGW:
winget install BrechtSanders.WinLibs.POSIX.UCRT

# 2. Build (build.ps1 puts cargo + WinLibs on PATH, compiles, and stages dist\):
.\build.ps1
```

Output: `dist\azure_sapi.dll` and `dist\setup.exe`. (`build.ps1` finds the WinLibs
MinGW automatically; with an MSVC toolchain you can also just run
`cargo build --release` inside `azure-sapi\`.)

## Command reference

```
setup install                     Register voices, save key/region, set default, test
setup uninstall [--purge]         Remove voices (and assets dir with --purge)
setup default-voice <Voice>       Set the active voice (and clear the OneCore override)
setup rate <0.5..2.0>             Set speech speed for all voices
setup test [Voice]                Speak a test sentence
```

Voices: `AzureThomas`, `AzureChristopher`, `AzureAria`, `AzureGuy`, `AzureJenny`.

Install reads `AZURE_SPEECH_REGION` and `AZURE_SPEECH_KEY` from the environment if
set, otherwise prompts for them.
